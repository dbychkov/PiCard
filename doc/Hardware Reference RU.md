# Аппаратная Архитектура PiCard — Техническое Введение

---

## Обзор Аппаратной Архитектуры

PiCard V3 — это **плата расширения, совместимая с шиной NemoBus** ZX Spectrum, вся активная логика которой реализована внутри единственного модуля **WeAct Studio RP2350B**. Там, где обычная клоновая плата содержала бы десятки дискретных ТТЛ/КМОП микросхем, ПЛМ или ПЛИС для эмуляции унаследованного набора периферийных чипов, PiCard заменяет их все программируемой матрицей GPIO RP2350, тремя автоматами состояний PIO, двумя DMA-подсистемами и программным Z80-ядром, выполняемым на Ядре 0.

Плата имеет два хорошо разделённых электрических домена:

| Домен | Напряжение | Устройства |
|-------|-----------|-----------|
| **ZX NemoBus / Шина Spectrum** | +5 В КМОП | Краевые разъёмы шины (J1/X1A, J2/X1B), 74HC245 (U1), 74HC04 (U2), транзисторы BC817 (Q1–Q4) |
| **Цифровая логика RP2350B** | +3,3 В КМОП | Модуль WeAct RP2350B, PCF8575 (U3), TDA1387T (U4), UART, SD-карта (SPI1) |

Местный **LDO-стабилизатор AMS1117-3.3 (U6)** формирует шину +3,3 В из источника питания +5 В, поступающего с краевого разъёма Spectrum. Конденсаторы C3, C10, C19 (по 10 мкФ каждый) обеспечивают местную развязку по шине +3,3 В; C20 (220 мкФ) — балластный накопительный конденсатор на шине +5 В. Три паяные перемычки (JP_5V0/JP_5V1/JP_5V2) позволяют независимо подключать или соединять зоны питания +5 В, обеспечивая гибкость при стендовых испытаниях без живого Spectrum.

**48 выводов GPIO** RP2350B разделены на пять функциональных групп:

1. **Сигналы шины ZX** — GPIO 1–7 (управление), GPIO 8–23 (A0–A15), GPIO 24–31 (D0–D7)
2. **Аудиовыход I2S** — GPIO 44 (DATA), GPIO 45 (BCLK), GPIO 46 (LRCK)
3. **SPI SD-карты** — GPIO 40–43
4. **Джойстик + UART** — GPIO 32–37 (совместно I2C/UART/DIV\_OFF)
5. **Клавиатура PS/2** — GPIO 38–39

## Описание Ключевых Компонентов

### Модуль WeAct Studio RP2350B

Модуль интегрирует микроконтроллер Raspberry Pi RP2350 и **8 МБ PSRAM** (APS6404L) на одной плате. PSRAM подключена непосредственно к контроллеру **QMI (QSPI Memory Interface)** RP2350 и отображена в память по адресу `0x11000000` через XIP CS1 (`SFE_RP2350_XIP_CSI_PIN = GPIO 0`). Эмуляция карты GS использует эту PSRAM для хранения до 62 банков × 16 КБ = **992 КБ** эмулируемой ОЗУ GS (массивы `RAM_BANK1`/`RAM_BANK2`, объявленные с атрибутом компоновщика `PSRAM`), что значительно превышает возможности любой платы на базе RP2040.

MCU разогнан до **351 МГц** через `vreg_set_voltage(VREG_VOLTAGE_1_60)` + `set_sys_clock_hz(351×10⁶, 1)`. Этот запас производительности необходим для одновременного выполнения программного Z80 на эквивалентной исходной тактовой частоте, синтеза двойного AY-звука каждые 9 мкс и обслуживания шины ZX 3,5 МГц в плотном цикле опроса.

### ЦАП I2S TDA1387T (U4)

**TDA1387T** (Philips/NXP) — это полный 16-битный стереофонический ЦАП I2S в корпусе SO-8. Он принимает стандартный последовательный поток Philips I2S (SCLK, LRCK, DATA) и формирует два фильтрованных аналоговых напряжения на выводах A\_LEFT (вывод 6) и A\_RIGHT (вывод 8).

**Автомат состояний PIO I2S (`audio_i2s_program`):**

8-инструкционная PIO-программа в `src/i2s/i2s.c` реализует протокол I2S полностью аппаратно, используя механизм **side-set** для одновременного переключения BCLK и LRCK с каждым битом данных:

```
side-set биты[1:0]: { LRCK, BCLK }

set  x, 14          side 3  ; BCLK=1 LRCK=1, установить счётчик на 15 бит
out  pins, 1        side 2  ; MSB левого семпла, BCLK=1 LRCK=0
jmp  x--, prev      side 3  ; ещё 14 левых бит...
out  pins, 1        side 0  ; последний левый бит, BCLK=0 LRCK=0
set  x, 14          side 1  ; BCLK=0 LRCK=1
out  pins, 1        side 0  ; MSB правого семпла...
jmp  x--, prev      side 1  ; ещё 14 правых бит...
(wrap)
```

Сдвиговый регистр настроен на **32-битный автоматический pull, MSB первым** (`sm_config_set_out_shift(&c, false, true, 32)`), поэтому один вызов `i2s_out(L, R)` упаковывает оба 16-битных семпла в одно 32-битное слово и записывает его в TX FIFO PIO.

Делитель тактовой PIO вычисляется как:
```c
uint32_t divider = clock_get_hz(clk_sys) * 4 / (44100 * 8);
```
Это генерирует ровно 44 100 фронтов LRCK в секунду независимо от разгона CPU до 351 МГц.

**Цепочка DMA для аудиовывода без участия CPU:**  
`i2s_init()` настраивает два цепочечных DMA-канала. Первичный канал (`dma_i2s`) многократно копирует единственное слово `i2s_data` в TX FIFO PIO, используя сигнал DREQ PIO для управления темпом передач. Вторичный канал (`dma_i2s_ctrl`) перезагружает счётчик передач первичного канала по завершении, создавая бесконечный цикл пакетов по 1024 слова. Итог: функции `i2s_out()` достаточно обновить единственную переменную `uint32_t`; всё остальное делает аппаратура.

**Вывод фильтра TDA1387T:**  
Вывод FLT (вывод 7) подключён к внешней RC-цепи (C15 = 0,1 мкФ и сопутствующие элементы), которая задаёт частоту среза внутреннего фильтра восстановления 4-го порядка, подавляя образы квантования I2S выше аудиополосы.

### I2C-расширитель ввода-вывода PCF8575DBR (U3)

**PCF8575** предоставляет 16 квазидвунаправленных выводов ввода-вывода (P0–P7 и P10–P17) на одном ведомом I2C по адресу `0x20` (выводы A0=A1=A2=GND). Он непрерывно опрашивается Ядром 0 на частоте 400 кГц через I2C0 (GPIO 32/33).

**Подключение джойстика:**  
Выводы порта P0–P8 подключаются к J10 (JOY-1, 9-контактный DE-9 разъём стандарта Atari/Kempston) через подтягивающие резисторы 10 кОм (R20–R28). При отсутствии джойстика все линии притянуты к высокому уровню, давая состояние покоя `0xFF`. При нажатии направления или кнопки огня соответствующий P-вывод замыкается на GND внутренним переключателем джойстика.

Порт расширяется на J9 (блок вторичного разъёма) для возможного второго джойстика на P10–P17 с той же схемой подтяжки.

**Неблокирующая передача I2C:**  
Функция прошивки `i2c_transfer_non_blocking()` (`src/i2c/i2c_joy.cpp`) обходит блокирующий API SDK (`i2c_write_blocking` / `i2c_read_blocking`). Вместо этого она:
1. Читает предыдущий результат непосредственно из аппаратного RX FIFO I2C.
2. Загружает новую команду чтения в TX FIFO с `I2C_IC_DATA_CMD_CMD_BITS | STOP`.

Этот пинг-понговый подход завершается за несколько тактов CPU и сразу возвращает управление, поэтому аудиотик Ядра 0 в 9 мкс никогда не блокируется в ожидании транзакции I2C.

**Преобразование кнопок в формат Kempston:**  
`joy_proc()` вызывает `i2c_transfer_non_blocking()` на каждом тике и конвертирует сырое 16-битное показание PCF8575 в формат Kempston с помощью предварительно вычисленной таблицы из 256 элементов (`nes_to_kempston_convert[]` или `sega_to_kempston_convert[]`). Байт Kempston сохраняется в `joy_state` и выдаётся Spectrum при чтении порта ввода-вывода `0x1F` Ядром 1.

**Светодиодный индикатор:**  
D6 (LED) с R46/R47 (по 10 кОм) подключён к выводу `~INT` PCF8575, обеспечивая визуальную индикацию активности I2C.

### Шинный приёмопередатчик 74HC245 (U1) и инверторы 74HC04 (U2)

#### 74HC245 — Двунаправленный буфер шины данных

U1 — критический интерфейс между **шиной данных ZX Spectrum 5 В** (выводы B-стороны B0–B7, подключённые к краевому разъёму NemoBus) и **линиями данных RP2350B 3,3 В** (выводы A-стороны A0–A7, подключённые к GPIO 24–31). Питается от шины +5 В.

| Вывод 74HC245 | Цепь | Сигнал |
|---------------|------|--------|
| 1 (DIR) | RDR/ | Управление направлением — управляется транзистором Q1 от GPIO 2 |
| 19 (CE/) | GND / коллектор транзистора | Разрешение микросхемы (активный низкий уровень) |
| 2–9 (A0–A7) | GPIO 24–31 | Сторона 3,3 В (RP2350B) |
| 11–18 (B0–B7) | \*D0–\*D7 | Шина данных NemoBus 5 В |
| 20 (VCC) | +5 В | Питание |

**Управление направлением от side-set PIO (GPIO 2 = `PIN_DATA_CTRL`):**

PIO-программа драйвера шины (`src/PIO/pio_data_z80_bus.cpp`) выделяет свой 1-битный side-set для GPIO 2. Транзисторная схема вокруг Q1 (BC817, делители базы R23/R27, коллекторная нагрузка R29 к +5 В) выполняет две задачи:

1. **Сдвиг уровней**: преобразует выходной сигнал GPIO 2 на 3,3 В в логический сигнал 5 В, пригодный для управления выводами DIR/CE 74HC245, работающими при порогах КМОП 5 В.
2. **Инверсия полярности**: коллектор NPN логически инвертирован — GPIO 2 HIGH → Q1 насыщён → коллектор притянут к LOW.

PIO-программа использует это для реализации атомарной последовательности «предзагрузка, затем включение»:

```
шаг 0: mov pindirs, null   side 1  → GPIO2=1 → Q1 вкл. → DIR LOW (B→A, Spectrum→RP2350) → GPIO24-31 = входы
шаг 1: pull block          side 1  → ожидание данных в TX FIFO
шаг 2: jmp pin,0           side 1  → прервать, если /RD не утверждён (нет активного цикла чтения)
шаг 3: out pins, 8         side 1  → загрузить выходной байт в регистр GPIO24-31 (ещё входы)
шаг 4: mov pindirs, ~null  side 0  → GPIO2=0 → Q1 выкл. → DIR HIGH (A→B, RP2350→Spectrum) → GPIO24-31 = выходы
шаг 5: jmp pin,0           side 0  → /RD снят? → вернуться к шагу 0
шаг 6: jmp 5               side 0  → удерживать шину активной пока /RD всё ещё низкий
```

Шаги 3 и 4 — критическая пара: байт записывается в **регистр выхода** вывода, пока направление ещё INPUT, затем направление переключается в OUTPUT за один такт PIO (~2,8 нс при 351 МГц). Это исключает любой глюч, при котором неверное значение могло бы появиться на шине хотя бы на один такт.

#### 74HC04 — Шестикратный инвертор (U2)

U2 (74HC04, питание от +5 В) обеспечивает несколько одиночных инверсий для:

- **Сдвиг уровней NMI и RESET**: выходы GPIO 35 (/NMI) и GPIO 47 (/RESET) на 3,3 В слишком слабы для непосредственного утверждения линий с открытым коллектором 5 В против подтяжек шины Spectrum. U2E/U2F инвертируют и буферизуют эти сигналы; диоды D1–D3 (1N4148W) обеспечивают ограничение и изоляцию для предотвращения обратной подачи тока.
- **Маршрутизация ROM\_BLK**: GPIO 1 (BLK) проходит через вентиль 74HC04 перед управлением внешней схемой буфера блокировки ПЗУ, гарантируя соответствие требованиям по управлению 5 В.
- **Дребезг SW1/SW2**: кнопка NMI (SW1) и кнопка RESET (SW2) проходят через два последовательных вентиля 74HC04 с RC-цепями (конденсаторы развязки 10 мкФ на выходах переключателей) для аппаратного устранения дребезга перед поступлением сигнала на GPIO RP2350B или шину.

**Почему 74HC04, а не специализированный сдвигатель уровней?**  
На скорости шины Spectrum 3,5 МГц задержка распространения вентиля 74HC04 (~7 нс типичное при 5 В) пренебрежимо мала. Использование стандартного логического вентиля с известным гистерезисом ±0,8 В (порог КМОП) проще и дешевле, чем двунаправленный сдвигатель уровней для этих однонаправленных сигналов управления.

### Операционные усилители MCP602 (U5A, U5B, U5C)

Два двойных операционных усилителя **MCP602** с входом и выходом до шины питания реализуют **аналоговый каскад после ЦАП** между аналоговыми выходами TDA1387T и стереофоническим разъёмом для наушников PJ325 (J5).

**Тракт сигнала — правый канал (U5A):**

```
TDA1387T A_RIGHT (вывод 8)
    → R39 / R40 (10 кОм)       Разделение по переменному току / задание входного импеданса
    → C12 (10 мкФ, электролит.) Конденсатор блокировки постоянного тока (ФВЧ: fc ≈ 1,6 Гц)
    → U5A (+) неинвертирующий вход (вывод 3)
    → U5A выход (вывод 1)
    → R41 (62 кОм) ОС на U5A (−) вход (вывод 2)
         → C16 (470 пФ)         Спад ВЧ в цепи ОС: fc = 1/(2π·62k·470p) ≈ 5,5 кГц
    → R36 (620 Ом)              Защитный последовательный резистор выхода (КЗ / ёмкостная нагрузка)
    → J4 (out_A, канал R)
    → J5 (PJ325, Ring — правый)
```

**Тракт сигнала — левый канал (U5B):**

```
TDA1387T A_LEFT (вывод 6)
    → R40 / C11 (10 мкФ)         тот же каскад блокировки постоянного тока
    → U5B (+) вход (вывод 5)
    → U5B выход (вывод 7)
    → R42 (62 кОм) → C17 (470 пФ)  тот же ВЧ-фильтр
    → R35 (620 Ом)
    → J3 (in_A, канал L)
    → J5 (PJ325, Tip — левый)
```

**U5C** — это вывод питания второго корпуса MCP602 (V− = вывод 4, V+ = вывод 8), развязанный конденсаторами C9 (10 мкФ) и C4 (0,1 мкФ). Усиления сигнала здесь нет.

**Расчёт коэффициента усиления:**  
В данной неинвертирующей конфигурации с равными резисторами обратной связи и заземления коэффициент усиления замкнутой петли:

$$
A_v = 1 + \frac{R_{ОС}}{R_{вх}}
$$

При R41 = R42 = 62 кОм в качестве резистора обратной связи и внутреннем выходном сопротивлении TDA1387T (~1 кОм типичное) в качестве эффективного `R_вх`, коэффициент усиления приблизительно равен единице (буфер ≈1) — основная функция — **преобразование импеданса** (высокоимпедансный выход TDA1387T → низкоимпедансный выход для наушников) и **подавление ВЧ-шума** через конденсатор 470 пФ в цепи обратной связи.

**Полюс 5,5 кГц** от R41–C16 является фильтром восстановления; поскольку RP2350B синтезирует звук с частотой ~111 кГц (период семпла 9 мкс), децимированной до 44,1 кГц на TDA1387T, образы на кратных 44,1 кГц уже ослабляются внутренним фильтром восстановления TDA1387T 4-го порядка. Каскад на операционном усилителе добавляет дополнительный спад 1-го порядка в качестве запаса по безопасности.

## Сопряжение с Шиной NemoBus ZX Spectrum

### Краевые Разъёмы

Плата несёт два **31-позиционных краевых разъёма PCB** (J1/X1A и J2/X1B), которые сопрягаются с портом расширения ZX Spectrum. Разъём охватывает обе стороны PCB (сторона A = X1A = J1, сторона B = X1B = J2), обеспечивая доступ ко всем 31 × 2 = 62 контактам NemoBus. Ключевые линии шины от краевого разъёма:

| Сигнал NemoBus | Подключение |
|----------------|------------|
| A0–A15 | GPIO 8–23 (через резисторную сборку RP1/RP2) |
| D0–D7 | B-сторона 74HC245 U1 (5 В); A-сторона → GPIO 24–31 |
| /MREQ, /IORQ, /RD, /WR, /M1 | GPIO 7, 6, 5, 4, 3 (через RP1/RP2) |
| /NMI | Коллектор Q3 BC817 с открытым коллектором (активный низкий уровень) |
| /RESET | GPIO 47 через Q2 BC817 (открытый коллектор) |
| IORQGE | J8 (вспомогательный разъём NemoBus) |
| BLK (блокировка ПЗУ) | GPIO 1 через Q4 BC817 → буфер 74HC04 |
| +5 В, GND | Вход AMS1117-3.3, балластные конденсаторы |

**Вспомогательный разъём J8 (Conn\_02x08\_Odd\_Even)** выводит специфические управляющие сигналы NemoBus, не имеющие прямых эквивалентов GPIO: `IORQGE`, `CSR/`, `DOS/`, `RDR/`, а также разводку для двух пар портов PS/2 и адресных линий краевого разъёма ZX A13, A15, B4, A25.

### Стратегия Сдвига Уровней 5 В → 3,3 В

Выводы GPIO RP2350 рассчитаны на **максимум 3,3 В** (абсолютный максимум 3,63 В). NemoBus работает на уровне ТТЛ/КМОП 5 В. PiCard V3 использует подход **массива ограничивающих ток резисторов**:

1. **RP1 и RP2** — компоненты резисторных сборок, установленные последовательно между краевым разъёмом и выводами GPIO RP2350B, обеспечивающие индивидуальные последовательные защитные резисторы (значения, помеченные «10», означают 10 кОм в контексте схемы) для каждой адресной и управляющей линии.

2. Когда сигнал NemoBus 5 В управляет выводом GPIO через 10 кОм, внутренний ЗЗ-диод RP2350 (проводящий при V_вывод > V_IOVDD + 0,5 В) поглощает избыточный ток:

$$
I_{огр} = \frac{5\text{ В} - 3,8\text{ В}}{10\text{ кОм}} \approx 0,12\text{ мА}
$$

Это значительно ниже номинального безопасного тока ограничения для выводов RP2350.

3. **R1–R8 и R20–R21** (также 10 кОм) — отдельные последовательные резисторы на конкретных управляющих сигналах (/M1, /WR, /RD, /IORQ, /MREQ) для той же цели.

4. **Шина данных (D0–D7):** Защищена самим 74HC245 — A-сторона U1 управляется RP2350B на 3,3 В. Когда 74HC245 работает в режиме B→A (Spectrum читает из RP2350), RP2350 управляет выходом 3,3 В через буфер; уровень HIGH выхода 74HC245 в этом направлении ограничен 3,3 В управляющей стороной RP2350. В режиме A→B (RP2350 читает шину Spectrum) 74HC245 транслирует сигналы 5 В Spectrum на свою сторону 5 В A, но B-сторона (RP2350) видит ослабленный уровень через выходной каскад микросхемы.

### Критические Управляющие Сигналы для RP2350B

Цикл мониторинга шины в `ZX_bus_task()` (Ядро 1) считывает **все 48 GPIO одновременно** одной инструкцией `gpio_get_all()` и извлекает нужные сигналы с помощью битовой маскировки:

```c
uint32_t datain32 = gpio_get_all();
uint32_t io_mrq_rd_wr = (~datain32) & (MREQ_Z80 | IORQ_Z80 | RD_Z80 | WR_Z80);
```

Активные низкие сигналы инвертируются программно, чтобы оператор `switch(io_mrq_rd_wr)` напрямую соответствовал четырём возможным типам цикла шины. 16-битный адрес извлекается как:

```c
uint16_t addr16 = (datain32 >> PIN_A0);  // сдвиг вправо на 8, даёт A0-A15 в битах [0:15]
```

Байт данных повторно семплируется непосредственно перед использованием (`gpio_get_all() >> PIN_D0`), а не доверяет начальному снимку, гарантируя корректность байта данных, даже если он установился немного позже управляющих линий:

```c
uint8_t d8 = gpio_get_all() >> PIN_D0;
```

### /NMI и /RESET как Выходы с Открытым Коллектором

Линии `/NMI` и `/RESET` на шине ZX должны быть **с открытым коллектором/стоком** — RP2350B не может принудительно выставить 5 В на эти линии и никогда не должен вступать в конфликт с внутренними подтяжками Spectrum. Прошивка реализует это корректно:

**NMI (`PIN_NMI = GPIO 35`):**
```c
void NMI_press() {
    gpio_put(PIN_NMI, 0);
    gpio_set_dir(PIN_NMI, GPIO_OUT);   // управлять низким уровнем → утвердить NMI
    busy_wait_ms(50);
    gpio_set_dir(PIN_NMI, GPIO_IN);    // высокий импеданс → освободить NMI
}
```
GPIO 35 управляет базой транзистора BC817 (через базовый резистор 10 кОм R25). Коллектор транзистора, подтянутый к +5 В через R30 (300 Ом), управляет линией `/NMI` NemoBus. Когда GPIO 35 выдаёт LOW, транзистор проводит, притягивая `/NMI` к почти GND. Когда GPIO 35 переведён в высокоимпедансный режим (вход), транзистор выключён и линия шины через R30 и собственную подтяжку Spectrum плавает к +5 В.

**ROM\_BLK (`PIN_ROM_BLK = GPIO 1`):**  
Аналогично GPIO 1 управляет Q4 (BC817), коллектор которого, нагруженный через буферный каскад 74HC04, управляет активной низкой линией **BLK**. Эта линия включает внешний буферный IC, физически блокирующий встроенное ПЗУ Spectrum от ответа на циклы MREQ, позволяя PiCard подставлять собственный образ ПЗУ (ПЗУ DivMMC ESXDOS, `ROM_DIVMMC[]`).

## Физические Интерфейсы и Соединения

### Аудиовыход — Стереоразъём PJ325 (J5)

Путь аудиосигнала от прошивки до разъёма для наушников:

```
Ядро 0 (каждые 9 мкс):
  mix_sound_out()
    → i2s_out(L, R)
      → упаковывает 32-битное слово в переменную i2s_data
         → DMA передаёт в TX FIFO PIO2
            → Автомат PIO2 тактирует биты → GPIO44/45/46
               → TDA1387T (U4) принимает поток I2S, декодирует в аналог
                  → выводы A_LEFT / A_RIGHT (±500 мВ прибл.)
                     → C12/C11 (10 мкФ, блокировка постоянного тока), R40/R39 (10 кОм вход)
                        → MCP602 U5A (правый), U5B (левый) — каскад буфера/фильтра
                           → R36/R35 (620 Ом защита выхода)
                              → J5 PJ325 стереоразъём 3,5 мм (Tip=L, Ring=R, Sleeve=GND)
```

PJ325 — стандартный разъём TRS 3,5 мм (J5, обозначен «Audio OUT»). R на схеме рядом с разъёмом — правый канал, L — левый. Выходное сопротивление, представленное наушникам, составляет приблизительно 620 Ом (R36/R35) плюс практически нулевое выходное сопротивление операционного усилителя — пригодно для непосредственного управления наушниками 32 Ом.

Постоянное смещение на выходе блокируется электролитическими конденсаторами связи C12/C11 (10 мкФ, рейтинг ≥10 В) на выходе TDA1387T.

Цепь +5VA (вывод 5 TDA1387T / питание VA) фильтруется отдельно конденсатором C8 (1 мкФ), а R36 образует П-фильтр с C8 для изоляции цифрового шума от аналогового питания.

### SD-Карта — Схема с Двумя Разъёмами

Два разъёма используют **одну шину SPI1** (GPIO 40=MISO, 42=CLK, 43=MOSI), но одну линию chip-select (SD\_CS0 = GPIO 41):

| Разъём | Тип | Применение |
|--------|-----|-----------|
| **J7 (Conn\_SD0)** | 6-контактный разъём шага 2,54 мм | Межплатное соединение; маршрутизирует сигналы SD на внешний разъём или дочернюю плату с полноразмерным гнездом SD |
| **J\_SD1 (Micro\_SD\_Card)** | Гнездо Micro-SD карты | Прямое гнездо Micro-SD на плате |

J7 несёт: SD\_CS0, SD\_MOSI, SD\_CLK, SD\_MISO, +3,3 В, GND — автономное 6-проводное SPI-SD-соединение. Оба разъёма заполнены, но поскольку они используют единственную линию CS, **одновременно должна быть вставлена только одна SD-карта** при нормальной работе. Конструкция позволяет переключаться между встроенным гнездом Micro-SD и подключением проводом к внешнему гнезду для различных корпусов.

**Конфигурация SPI1** (из прошивки):
```c
spi_init(SD_SPI, SD_SPI_FREQ_CLK);   // 10 МГц
```
Данные передаются через прямой доступ к регистру (`spi_get_hw(SD_SPI)->dr`), а не через DMA SDK, обеспечивая однобайтовую задержку для обработчиков портов DivMMC/ZC-SD в цикле шины Ядра 1.

C7 (47 мкФ) и C6 (100 мкФ) обеспечивают балластную развязку питания SD-карты +3,3 В, поглощая токовые выбросы во время операций записи SD.

### Джойстик — J10 (JOY-1, DE-9)

J10 — **9-контактный DE-9 порт джойстика, совместимый с Atari/Kempston**. Распиновка соответствует стандартному соглашению Atari:

| Вывод J10 | Сигнал | Вывод PCF8575 | Бит Kempston |
|-----------|--------|---------------|--------------|
| 1 | UP | P0 | бит 3 |
| 2 | DOWN | P1 | бит 2 |
| 3 | LEFT | P2 | бит 1 |
| 4 | RIGHT | P3 | бит 0 |
| 5 | POT Y | — | — |
| 6 | FIRE | P4 | бит 4 |
| 7 | +5 В | — | — |
| 8 | GND | — | — |
| 9 | POT X | P5 | — |

Подтягивающие резисторы R20–R28 (по 10 кОм) удерживают все входы PCF8575 на высоком логическом уровне при отсутствии джойстика. Кнопка джойстика замыкает соответствующий вывод на GND.

**Режим NES** прошивки (`NES_JOY_EN=1`) поддерживает серийные NES-геймпады, подключённые через порт PCF8575, используя последовательность latch/clock, которая сдвигает 8 состояний кнопок через 3 вывода GPIO (P1=CLK, P2=LATCH, P3=DATA через порт PCF8575). Результат транслируется через `nes_to_kempston_convert[256]` и выдаётся на порт Kempston `0x1F`.

### Клавиатура PS/2 — J6 (Mini-DIN-6)

J6 — **6-контактный разъём Mini-DIN** (стандарт PS/2):

| Вывод Mini-DIN | Сигнал | GPIO |
|----------------|--------|------|
| 1 | KB DATA1 | 39 (PIN\_PS2\_DATA) |
| 3 | GND | GND |
| 4 | +5 В | +5 В (через R14 1 кОм ограничение тока) |
| 5 | KB CLK1 | 38 (PIN\_PS2\_CLK) |

R12 и R13 (по 1 кОм) — **последовательные защитные резисторы** в линиях CLK и DATA между разъёмом Mini-DIN и выводами GPIO. Они ограничивают ток ЭСЗ-разряда, а также ограничивают ток, когда устройство PS/2 с питанием 5 В управляет GPIO-выводом 3,3 В. В сочетании с внутренними ЗЗ-диодами RP2350B (ограничение при ≈3,8 В) они обеспечивают безопасность GPIO-входов:

$$
I_{огр} = \frac{5\text{ В} - 3,8\text{ В}}{1\text{ кОм}} = 1,2\text{ мА}
$$

Это приемлемо для кратковременных ЭСЗ-событий; в установившемся режиме линии PS/2 с открытым коллектором подтянуты к высокому уровню через собственные подтяжки ~10 кОм клавиатуры, поэтому установившийся ток GPIO близок к нулю.

**Вторичный порт PS/2** (PS2\_CLK2 = GPIO 36, PS2\_DATA2 = GPIO 37) разделяет маршрутизацию выводов разъёма Mini-DIN через вспомогательный разъём NemoBus J8. Эти выводы перекрываются с UART1 (TX/RX). На этапе компиляции активна только одна функция: `HW_UART_EN=1` (по умолчанию) занимает GPIO 36/37 для UART1, второй порт PS/2 не используется. Переключение на `KB_PS2_EN` для альтернативного порта потребует перекомпиляции.

**Автомат состояний PIO0 для приёма PS/2:**  
Линия CLK PS/2 (GPIO 38) настроена как вывод JMP PIO. 5-инструкционная PIO-программа тактирует 11 бит за кадр (старт + 8 бит данных + чётность + стоп) с любой скоростью, которую обеспечивает клавиатура (обычно 10–16 кГц). Тактовая PIO делится до ≈200 кГц (`fdiv = clock_get_hz(clk_sys) / 200000`) для обеспечения чистого семплирования фронтов. Два цепочечных DMA-канала передают принятые 32-битные слова из RX FIFO PIO в 100-элементный кольцевой буфер (`PS_2_BUS[100]`) **без участия CPU** — Ядро 0 проверяет заголовок кольцевого буфера только каждые ~576 мкс для декодирования новых сканкодов.

### Ручное Управление — SW1 (NMI), SW2 (RESET)

| Кнопка | Функция | Путь GPIO |
|--------|---------|-----------|
| **SW1 (SW\_NMI)** | Принудительное NMI Z80 для Spectrum | SW1 → 74HC04 (U2F) → GPIO 39, затем к драйверу транзистора NMI |
| **SW2 (SW\_RST)** | Утверждение /RESET Spectrum на 500 мс | SW2 → 74HC04 (U2D/U2E) → GPIO 47 → Q2 → линия /RESET на шине |

Обе кнопки проходят через каскады инверторов 74HC04 (которые одновременно устраняют дребезг и обеспечивают совместимость порогов КМОП 5 В→3,3 В) перед поступлением на GPIO RP2350B или транзистор шины. R9 и R10 (10 кОм) обеспечивают подтяжки на входах переключателей.

В прошивке функция `NMI_press()` также может вызываться клавишей **Insert** клавиатуры PS/2, а комбинация **Ctrl+Alt+Del** вызывает `reset_spectrum()`, которая утверждает `/RESET` на 500 мс через GPIO, точно воспроизводя аппаратную функцию SW2.

## Сводка Распределения Питания

```
Краевой разъём ZX Spectrum (J1/X1A, J2/X1B)
    +5 В ─────────────────────────────── Перемычки JP_5V0 / JP_5V1 / JP_5V2
                │
                ├── C20 (220 мкФ балласт) ── GND
                │
                ├── D5 (светодиод питания, R43 300 Ом)
                │
                ├── U6 AMS1117-3.3
                │       VI (вход +5 В)
                │       VO (выход +3,3 В) ── C19, C3, C10 (по 10 мкФ, развязка)
                │               │
                │               ├── WeAct RP2350B (IOVDD, DVDD)
                │               ├── VCC PCF8575 (вывод 24)
                │               ├── VDD SD-карты (J_SD1 вывод 4, J7 вывод 5)
                │               └── TDA1387T VA (+5VA после фильтра R32, C7)
                │
                ├── VCC 74HC245 (вывод 20) ── +5 В (логика стороны шины)
                ├── VCC 74HC04 (вывод 14) ── +5 В
                └── J10 JOY-1 вывод 7 ── +5 В (питание джойстика)
```

AMS1117-3.3 рассчитан на выходной ток 1 А, что значительно превышает суммарную нагрузку RP2350B (~100 мА @ 351 МГц), PCF8575 (~2 мА) и TDA1387T (~5 мА).
